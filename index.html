📌 Шаг 1: Выбор редактора кода

🎯 Зачем нужен редактор кода?

Редактор — это программа, в которой ты пишешь код. Можно использовать обычный Блокнот, но в нём нет важных функций:

  • Подсветка синтаксиса — теги, атрибуты и значения раскрашиваются в разные цвета. Глаза не устают, ошибки видно сразу.
  • Автодополнение — начинаешь писать <div, а редактор сам подсказывает и закрывает тег.
  • Подсказки — наводишь на свойство CSS — показывает, что оно делает.
  • Расширения — можно добавить Live Server, проверку ошибок, красивую тему.

💡 Хороший редактор ускоряет работу в 3-5 раз и избавляет от глупых ошибок.

⭐ Visual Studio Code

Плюсы: бесплатно, тысячи расширений, встроенный терминал, Live Server, Git, подсветка всего подряд, русский интерфейс.

Минусы: требует 4-8 ГБ ОЗУ, запускается 2-3 секунды.

🔗 Скачать →

⚡ Sublime Text

Плюсы: очень быстрый (мгновенный запуск), лёгкий (меньше 50 МБ), красивая подсветка.

Минусы: часть функций платная (периодически просит купить лицензию за $99).

🌐 Онлайн-редакторы

CodePen, Replit, JSFiddle, GitHub Codespaces — работают прямо в браузере, не нужно ничего устанавливать.

Минусы: нужен интернет, ограниченный функционал в бесплатной версии.

✅ Мой совет для начинающего: Установи VS Code. Это стандарт в веб-разработке, по нему больше всего гайдов на YouTube и в текстовых уроках. Если у тебя слабый компьютер — используй Sublime Text или онлайн-редактор Replit.

📌 Шаг 2: Установка VS Code и расширений

🎯 Пошаговая инструкция по установке VS Code:

  1. Скачай VS Code с официального сайта: code.visualstudio.com → нажми синюю кнопку "Download for Windows" (или для твоей ОС).
  2. Запусти установщик — откроется окно установки. Нажимай "Далее" → "Я принимаю условия" → "Установить".
  3. После установки открой VS Code (иконка появится на рабочем столе или в меню "Пуск").
  4. Установи русский язык (опционально): нажми Ctrl+Shift+P → введи "Configure Display Language" → выбери "Russian" → перезапусти VS Code.

🔌 Какие расширения установить (самые важные):

  • Live Server — создаёт локальный сервер и автоматически обновляет страницу при сохранении файла. Как установить: нажми на иконку "Расширения" (четыре квадратика слева) → введи "Live Server" → нажми "Install".
  • Prettier — автоматически форматирует код: расставляет отступы, переносы строк, кавычки. Код становится красивым и единообразным.
  • Material Icon Theme — добавляет красивые иконки для файлов и папок.
  • Russian Language Pack — переводит интерфейс VS Code на русский.
// Как пользоваться Live Server:

💡 Полезный совет: В VS Code можно открывать несколько проектов одновременно в разных окнах. Также можно установить тему оформления — зайди в "Расширения", найди "One Dark Pro" или "Dracula Official" — глаза меньше устают.

📌 Шаг 3: Создание структуры проекта

🎯 Зачем нужна структура папок?

Когда проект маленький (один index.html) — можно всё хранить в одной папке. Но когда появляются стили, скрипты, картинки, шрифты — без структуры начинается хаос. Правильная организация помогает:

  • Быстро находить нужный файл
  • Подключать файлы по понятным путям (например, css/style.css)
  • Легко передавать проект другому разработчику
  • Подготавливать проект к публикации на хостинге

📁 Как создать структуру в VS Code:

  1. Открой VS Code
  2. Нажми "Файл" → "Открыть папку" → создай новую папку с названием мой-сайт → выбери её
  3. В левой панели (проводник) нажми на иконку "Новая папка" — создай папку css
  4. Создай папку js
  5. Создай папку images
  6. Нажми на иконку "Новый файл" — создай index.html в корне
  7. В папке css создай файл style.css
  8. В папке js создай файл script.js
мой-сайт/               # корневая папка проекта
│
├── index.html          # главная страница (всегда должен быть)
│
├── css/                # папка для стилей
│   └── style.css       # файл со всеми стилями сайта
│
├── js/                 # папка для скриптов
│   └── script.js       # файл с JavaScript-кодом
│
└── images/             # папка для картинок
    └── (сюда кидай .jpg, .png, .svg, .webp)

💡 Важно: Имена файлов и папок пиши только латиницей (английские буквы) и без пробелов. Вместо пробела используй дефис: my-photo.jpg или нижнее подчёркивание: my_photo.jpg. Это нужно, чтобы сайт корректно работал на любом хостинге.

📌 Шаг 4: HTML — скелет сайта

🎯 Что такое HTML?

HTML (HyperText Markup Language) — это язык разметки. Он не программирует, а описывает структуру страницы: где заголовок, где абзац, где картинка, где ссылка. Браузер читает HTML и показывает пользователю красиво оформленную страницу.

🧱 HTML состоит из тегов. Тег — это команда для браузера, обычно выглядит так: <название_тега>содержимое</название_тега>. Например, <h1>Заголовок</h1>.

📝 Базовый шаблон index.html (объяснение каждой строки):

<!DOCTYPE html>      👈 говорим браузеру "это HTML5, не переживай"
<html lang="ru">     👈 корневой тег, указываем язык страницы (ru — русский)
<head>               👈 служебная информация (не видна на странице)
    <meta charset="UTF-8">   👈 поддержка русских букв и иероглифов
    <meta name="viewport" content="width=device-width">  👈 адаптивность на телефонах
    <title>Мой первый сайт</title>  👈 текст на вкладке браузера
    <link rel="stylesheet" href="css/style.css">  👈 подключаем CSS
</head>
<body>               👈 всё, что внутри, видит пользователь
    <h1>Заголовок первого уровня</h1>  👈 самый главный заголовок (один на странице)
    <p>Это абзац текста. Здесь можно написать много слов.</p>
    <a href="https://example.com">Это ссылка</a>  👈 гиперссылка
    <img src="images/photo.jpg" alt="описание картинки">  👈 вставка картинки
    <script src="js/script.js"></script>  👈 подключаем JavaScript (обычно в конце body)
</body>
</html>

📚 Самые важные теги для начинающего:

  • <h1>...</h1> до <h6>...</h6> — заголовки (h1 — самый важный, h6 — самый мелкий)
  • <p>...</p> — абзац текста
  • <a href="ссылка">...</a> — ссылка
  • <img src="путь" alt="описание"> — картинка
  • <ul><li>...</li></ul> — маркированный список
  • <ol><li>...</li></ol> — нумерованный список
  • <div>...</div> — универсальный блок (контейнер)
  • <span>...</span> — строчный элемент для выделения части текста
  • <button>...</button> — кнопка

💡 Совет: Всегда закрывай теги! Если открыл <div>, обязательно закрой </div>. Незакрытые теги — частая причина, почему сайт "едет" или выглядит не так, как надо.

📌 Шаг 5: CSS — стилизация и дизайн

🎯 Что такое CSS?

CSS (Cascading Style Sheets) — это язык стилей. Он отвечает за внешний вид: цвета, шрифты, отступы, расположение элементов, анимации. Без CSS сайт выглядит как чёрно-белый документ из 90-х. С CSS — становится красивым и современным.

🧪 Попробуй прямо сейчас! Ниже ты можешь покрутить цвет и увидеть, как CSS меняет блок в реальном времени.

🎨 Покрути цвет — CSS меняется на глазах:

Этот блок меняет цвет! 
background: #4361ee;

💡 Вот так просто работает CSS! Ты меняешь одно свойство — блок сразу реагирует.

📝 Как писать CSS-правила:

/* Синтаксис: селектор { свойство: значение; } */
body {
    background-color: #0f172a;    /* цвет фона страницы */
    color: white;                 /* цвет текста */
    font-family: Arial, sans-serif; /* шрифт */
    margin: 0;                    /* убираем отступы по краям */
    padding: 20px;                /* внутренние отступы */
}

h1 {
    font-size: 48px;              /* размер шрифта */
    text-align: center;           /* выравнивание по центру */
    color: #4cc9f0;               /* ярко-голубой цвет */
}

.button {
    background: linear-gradient(135deg, #4cc9f0, #4361ee); /* градиент */
    border: none;                 /* убираем рамку */
    border-radius: 30px;          /* скругление углов */
    padding: 12px 24px;           /* внутренние отступы (вертикаль горизонталь) */
    cursor: pointer;              /* курсор-рука при наведении */
    transition: 0.3s;             /* плавность анимации */
}

.button:hover {
    transform: scale(1.05);       /* увеличивается на 5% при наведении */
    box-shadow: 0 0 20px #4cc9f0; /* свечение */
}

🔗 Как подключить CSS к HTML:

Внутри тега <head> добавь эту строку:

<link rel="stylesheet" href="css/style.css">

Важно: путь href="css/style.css" должен указывать на реальное расположение файла. Если style.css лежит в корне — пиши href="style.css".

💡 Советы по CSS:

  • Используй инспектор кода (F12 → вкладка Elements или Styles) — там можно менять CSS на лету и сразу видеть результат.
  • Для цветов используй hex-коды (#ffffff — белый, #000000 — чёрный) или названия цветов (red, blue, green).
  • Flexbox — лучший способ выравнивать элементы. Достаточно написать display: flex; justify-content: center; align-items: center; и элемент окажется по центру.

📌 Шаг 6: JavaScript — интерактивность

🎯 Что такое JavaScript?

JavaScript (JS) — это язык программирования, который делает сайт живым. HTML — это скелет, CSS — это одежда, а JavaScript — это мышцы и мозги. С помощью JS можно:

  • Реагировать на клики, наведение мыши, ввод текста
  • Менять стили и содержимое страницы без перезагрузки
  • Отправлять данные на сервер (формы, комментарии)
  • Создавать игры, анимации, калькуляторы, чаты

📝 Пример простого скрипта (script.js) с пояснениями:

// 1. Находим элемент на странице по тегу h1
const heading = document.querySelector('h1');

// 2. Добавляем обработчик события "клик"
heading.addEventListener('click', function() {
    // 3. Меняем цвет текста на красный, если был не красный, и наоборот
    if (heading.style.color === 'red') {
        heading.style.color = 'white';
    } else {
        heading.style.color = 'red';
    }
});

// 4. Функция, которая показывает всплывающее окно
function showMessage() {
    alert('Привет от JavaScript! 🎉');
}

// 5. Находим все кнопки на странице и добавляем им анимацию при клике
document.querySelectorAll('button').forEach(button => {
    button.addEventListener('click', function() {
        this.style.transform = 'scale(0.95)';
        setTimeout(() => {
            this.style.transform = 'scale(1)';
        }, 200);
    });
});

🔗 Как подключить JavaScript к HTML:

Добавь перед закрывающим тегом </body>:

<script src="js/script.js"></script>

Почему в конце body? Потому что браузер читает HTML сверху вниз. Если скрипт подключить в head, он может не найти элементы на странице (их ещё нет). В конце body — все элементы уже созданы.

💡 Полезные советы по JavaScript:

  • Открывай консоль браузера (F12 → вкладка Console). Там видны ошибки и можно писать JS-код вживую для тестов.
  • console.log('текст') — выводит сообщение в консоль. Очень полезно для отладки.
  • Не бойся ошибок! Это нормально. Консоль подскажет, что пошло не так.
  • Изучай события: click, mouseover, mouseout, keydown, submit — они самые нужные.

📌 Шаг 7: Адаптивный дизайн

🎯 Что такое адаптивность?

Адаптивный дизайн — это когда сайт одинаково хорошо выглядит и на компьютере (широкий экран), и на планшете, и на телефоне (узкий экран). Без адаптивности на телефоне текст будет мелким, а кнопки — крошечными.

📱 Сегодня более 60% трафика идёт с телефонов. Если сайт не адаптивен — пользователи уходят.

📝 Как сделать адаптивность через CSS-медиа-запросы:

/* Базовые стили для компьютеров (широкие экраны) */
.container {
    max-width: 1200px;
    margin: 0 auto;
    padding: 20px;
}

h1 {
    font-size: 48px;
}

/* Планшеты (ширина экрана до 768px) */
@media (max-width: 768px) {
    .container {
        padding: 15px;
    }
    h1 {
        font-size: 32px;
    }
}

/* Телефоны (ширина экрана до 480px) */
@media (max-width: 480px) {
    .container {
        padding: 10px;
    }
    h1 {
        font-size: 24px;
    }
    p {
        font-size: 14px;
    }
    /* Скрываем меню или делаем его вертикальным */
    nav {
        flex-direction: column;
    }
}

📱 Как проверить адаптивность в браузере:

  1. Открой свой сайт
  2. Нажми F12 (откроются инструменты разработчика)
  3. Нажми на иконку телефона Toggle device toolbar (или Ctrl+Shift+M)
  4. Выбери из списка iPhone, Pixel, iPad или введи свои размеры
  5. Смотри, как сайт выглядит на разных экранах

💡 Советы по адаптивности:

  • Используй относительные единицы: % (проценты), vw (ширина экрана), vh (высота экрана), rem (относительно корневого шрифта). Избегай фиксированных пикселей (px) для ширины.
  • Картинкам ставь max-width: 100%; height: auto; — тогда они не вылезут за границы экрана.
  • Для меню на телефоне используй бургер-меню (три полоски) — оно экономит место.
  • Кнопки на телефоне должны быть не меньше 44×44 пикселей — чтобы удобно нажимать пальцем.

📌 Шаг 8: Тестирование и отладка

🎯 Зачем тестировать сайт?

Даже опытные разработчики делают ошибки: опечатки в коде, неправильные пути к картинкам, забытые теги. Тестирование помогает найти и исправить эти ошибки до того, как сайт увидят пользователи.

🚀 Как запустить сайт для тестирования:

  1. Открой папку с проектом в VS Code
  2. Найди файл index.html в списке файлов
  3. Нажми на него правой кнопкой мыши
  4. Выбери "Open with Live Server" (если расширение установлено)
  5. Откроется браузер с адресом http://localhost:5500
  6. Теперь при каждом сохранении файла (Ctrl+S) сайт будет обновляться автоматически

🔍 Инструменты разработчика (DevTools) — F12:

  • Elements (Элементы) — смотришь HTML-код страницы в реальном времени, можешь менять текст и стили прямо в браузере (временно).
  • Console (Консоль) — здесь видны ошибки JavaScript. Если что-то не работает — сначала загляни сюда.
  • Network (Сеть) — показывает, загружаются ли картинки, CSS и JS файлы. Если файл красный — значит, путь неправильный.
  • Device Toolbar (Режим телефона) — иконка телефона сверху слева. Проверяй адаптивность.

💡 Что проверять перед публикацией:

  • ✅ Сайт открывается без ошибок в консоли (F12 → Console — пусто)
  • ✅ Все картинки загружаются (нет иконки "битой ссылки")
  • ✅ Все ссылки ведут туда, куда нужно
  • ✅ На телефоне (или в режиме эмуляции) текст читается, кнопки нажимаются
  • ✅ В разных браузерах (Chrome, Firefox, Edge) выглядит одинаково

📌 Шаг 9: Оптимизация перед публикацией

🎯 Что такое оптимизация и зачем она нужна?

Оптимизация — это ускорение сайта. Чем быстрее загружается сайт, тем лучше удержание пользователей. Google тоже любит быстрые сайты и ставит их выше в поиске.

⏱️ Идеальное время загрузки — до 2 секунд. Если дольше — люди уходят.

⚡ Что можно оптимизировать:

  • Изображения — самые тяжёлые файлы на сайте. Используй сервис TinyPNG — он сжимает PNG и JPG без потери качества (в 2-3 раза).
  • Форматы изображений — используй WebP вместо JPG/PNG. Он весит на 30% меньше. Можно конвертировать через CloudConvert.
  • CSS и JS — удали лишние пробелы, комментарии, переносы строк. Это называется "минификация". Сервис: minify-js.com.
  • Шрифты — не подключай 5 начертаний одного шрифта. Оставь обычный и жирный.

📊 Как проверить скорость сайта:

  • Google PageSpeed Insights — pagespeed.web.dev. Вставь ссылку на сайт, получишь оценку от 0 до 100 и список проблем.
  • GTmetrix — подробный отчёт с рекомендациями.
  • Lighthouse — встроен в браузер (F12 → вкладка Lighthouse → Generate report).

💡 Чек-лист перед публикацией:

  • ✅ Все картинки сжаты (TinyPNG)
  • ✅ Нет лишних пробелов в коде (Prettier помогает)
  • ✅ Сайт открывается на телефоне (проверь через эмуляцию)
  • ✅ Все ссылки рабочие (нет 404 ошибок)
  • ✅ Консоль браузера чистая (нет красных ошибок)

📌 Шаг 10: Публикация на GitHub Pages

🎯 Что такое GitHub Pages?

GitHub Pages — это бесплатный хостинг от компании GitHub. Туда можно загрузить свой сайт и получить ссылку, по которой сайт увидят все — друзья, учителя, будущие работодатели.

🌍 Твой сайт будет доступен по адресу: username.github.io/название-репозитория

📦 Пошаговая инструкция (10 минут):

  1. Зарегистрируйся на GitHub — github.com. Придумай логин (username), он будет в ссылке.
  2. Создай новый репозиторий — нажми зелёную кнопку "New" (или "+" → "New repository"). Назови его, например, my-first-site. Оставь Public. Нажми "Create repository".
  3. Загрузи файлы — на открывшейся странице нажми "uploading an existing file". Перетащи файлы: index.html, style.css, script.js, папки css/, js/, images/.
  4. Сохрани изменения — внизу страницы напиши комментарий "первая загрузка" и нажми зелёную кнопку "Commit changes".
  5. Включи GitHub Pages — зайди в "Settings" (вкладка вверху) → слева выбери "Pages". В разделе "Branch" выбери "main" (или "master") и нажми "Save".
  6. Подожди 2 минуты — GitHub соберёт сайт. Обнови страницу, появится ссылка: https://username.github.io/my-first-site/
  7. Проверь — перейди по ссылке. Твой сайт в интернете! 🎉

🔧 Что делать, если сайт не открывается или не видит стили:

  • Проверь, что файл называется именно index.html (с маленькой буквы). GitHub ищет именно его.
  • Проверь пути к файлам в HTML. Если стиль лежит в css/style.css, то в HTML должно быть <link rel="stylesheet" href="css/style.css"> (без первого слеша).
  • Имена файлов и папок должны быть на латинице и без пробелов.
  • Подожди ещё 5 минут — иногда GitHub Pages включается не сразу.

💡 Альтернативные бесплатные хостинги:

  • Netlify — просто перетаскиваешь папку с сайтом на сайт — и готово. Очень просто.
  • Vercel — удобно для современных проектов.
  • GitLab Pages — аналог GitHub Pages.
  • InfinityFree — бесплатный хостинг с PHP и базой данных.
